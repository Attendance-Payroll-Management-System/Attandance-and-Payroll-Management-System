# Salary Slip Email Feature - Implementation Plan

## Overview
Enhance the existing salary_slip.php page to generate PDF salary slips, attach them to emails via PHPMailer, and track email delivery history.

## Architecture

### Database Changes
**New table: `email_logs`** — tracks every email sent (individual or bulk)
```sql
CREATE TABLE email_logs (
    id INT PRIMARY KEY AUTO_INCREMENT,
    payroll_id INT NOT NULL,
    employee_id INT NOT NULL,
    recipient_email VARCHAR(100) NOT NULL,
    subject VARCHAR(255) NOT NULL,
    status ENUM('sent', 'failed', 'pending') DEFAULT 'pending',
    error_message TEXT DEFAULT NULL,
    sent_at DATETIME DEFAULT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (payroll_id) REFERENCES payrolls(id) ON DELETE CASCADE,
    FOREIGN KEY (employee_id) REFERENCES employee(id) ON DELETE CASCADE
);
```
**New migration file:** `config/migration_email_logs.sql`

### Files to Create
1. **`config/pdf_generator.php`** — PDF generation helper using FPDF library
2. **`vendor/FPDF/`** — Downloaded FPDF library (no Composer needed)
3. **`admin/email_history.php`** — New page to view/send/resend salary slips with full email history

### Files to Modify
1. **`admin/salary_slip.php`** — Major rewrite:
   - Generate PDF salary slip using PDF library
   - Attach PDF to email via PHPMailer (currently uses custom smtp_mailer which doesn't support attachments)
   - Log every send to `email_logs` table
   - Add "Bulk Send All" button
   - Add "Download PDF" button per employee
   - Show email status column (Sent/Failed/Pending)
   - Add Resend button for previously failed sends
   - Add Email History section at bottom
   - Email body matches requested format exactly

2. **`config/smtp_mailer.php`** — Enhance `smtp_mail()` to support PDF attachments (currently sends plain HTML only)

3. **`includes/sidebar.php`** — No change needed (salary_slip.php already in Payroll nav section)

4. **`config/db.sql`** — Add email_logs table definition

## Detailed Implementation Steps

### Step 1: Install FPDF Library
- Download FPDF from fpdf.org
- Place in `vendor/FPDF/` directory (fpdf.php + font files)
- No Composer required — just `require_once 'vendor/FPDF/fpdf.php'`

### Step 2: Create PDF Generator Helper (`config/pdf_generator.php`)
- Function: `generate_salary_slip_pdf($slip_data, $month_name, $year)` → returns PDF as string
- Renders a professional A4 salary slip with:
  - Company header (AURA HR branding with gradient)
  - Employee info (name, code, department)
  - Salary breakdown table (basic, OT, bonus, deductions, gross, net)
  - Period info
  - Footer with "PAID" badge
- Returns raw PDF content (for both download and email attachment)

### Step 3: Create Email Logs Migration (`config/migration_email_logs.sql`)
- Create the `email_logs` table
- Admin runs this SQL once to add the table

### Step 4: Rewrite `admin/salary_slip.php`
**Backend changes:**
- Single send: Generate PDF → attach to PHPMailer email → log to email_logs
- Bulk send: Loop through all payroll records with email → send each → log each
- Download: Generate PDF → stream as download (Content-Disposition: attachment)
- Email body text matches the exact format requested:
  ```
  Dear [Employee Name],
  
  Please find attached your salary slip for [Month] [Year].
  
  If you have any questions regarding your salary details, please contact the HR Department.
  
  Best regards,
  HR Department
  ```
- Subject: "Salary Slip for [Month] [Year]"
- Validate email before sending (PHP filter_var FILTER_VALIDATE_EMAIL)
- Track status: sent/failed/pending in email_logs

**Frontend changes:**
- Add "Download PDF" button per row
- Add "Send All" button in header area
- Add "Email Status" column (badge: Sent=green, Failed=red, Pending=yellow)
- Add "Resend" button for failed emails
- Add collapsible "Email History" section at bottom showing all past sends
- Use existing glass-strong / Tailwind patterns for UI consistency

### Step 5: Enhance `config/smtp_mailer.php`
- Add attachment support to `smtp_mail()`:
  - New parameter: `$attachments = []` (array of file paths or raw content)
  - Build MIME multipart message with boundaries
  - Encode attachments as base64
  - Set proper Content-Type headers for multipart/mixed

### Step 6: Update `config/smtp.php` Configuration
- Clean up the placeholder credentials
- Keep the same structure, just ensure proper defaults

## Key Design Decisions
- **FPDF over DOMPDF**: FPDF is lightweight, no Composer/Composer needed, works on any WAMP setup. DOMPDF would require Composer.
- **PDF generation on-demand**: PDFs are generated fresh each time (not stored on disk) to avoid storage management and ensure always up-to-date data.
- **PHPMailer over custom SMTP**: PHPMailer (already installed in vendor/) has built-in attachment support. We'll switch from the custom `smtp_mailer.php` to PHPMailer for reliability.
- **Email logs in DB**: Storing in MySQL allows querying, filtering, and displaying history without file I/O.
